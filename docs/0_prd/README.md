# 高性能三维建模软件 PRD

> 文档版本：v1.0（技术预研与立项版）  
> 更新日期：2026-06-19  
> 推荐架构：Node.js（NestJS）+ Electron + Vue 3 + Three.js + WebGPU/WebGL2 
> 结论等级：**有条件可行**

## 核心结论

本项目可行，但必须明确系统边界：

- **Three.js** 负责实时渲染、场景交互、显示代理与网格级编辑，不承担精确 CAD 拓扑内核职责。
- **NestJS** 负责项目管理、数据持久化、导入任务编排、缓存、命令日志、HTTP/SSE 与协议传输，不承担高成本 CAD 几何计算。
- **Electron** 负责桌面生命周期、安全边界、文件系统访问、进程监管、签名更新和操作系统集成。
- **原生几何侧车** 负责 STEP、DWG、高保真 DXF、B-Rep、精确测量、布尔/倒角等几何操作，以及高成本三角化（MVP阶段不需要实现）。
- **WebWorker 调度器** 负责 BVH、LOD、选择、轻量解析、压缩解码、点云过滤等前端 CPU 密集任务。
- **Protobuf 元数据 + 外置二进制分块** 作为图形数据协议主线；SSE 只传递进度、版本和资源失效等小事件。

## 推荐阅读顺序

1. 先阅读执行摘要、技术路线和总体架构。
2. 冻结 DWG SDK、精确建模边界和性能样本后，再确认产品范围。
3. 协议、命令事务、Worker 调度器与持久化模型应在 MVP 开发前完成设计评审。
4. 以 Phase 0 POC 的真实数据结果修订性能指标和里程碑。

## 关键立项门槛

- 确定 ODA Drawings SDK 与 Autodesk RealDWG 的商业、平台和分发路线。
- 确定 V1 的“建模”究竟是场景/网格编辑，还是包含精确 B-Rep 特征。
- 建立客户真实 DXF/DWG/STEP/PCD/GLB 样本库和基准硬件。
- 验证实体 ID 从源拓扑到三角网格、选取、命令和导出的稳定映射。
- 验证 Electron 同源本地服务下的 COOP/COEP、SharedArrayBuffer 和降级路径。
