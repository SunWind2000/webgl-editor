# 4. 产品范围、用户与核心流程

## 4.1 目标用户

| 用户 | 核心任务 |
|---|---|
| 工程审阅人员 | 打开多格式模型、测量、剖切、标注、对比版本、导出截图/报告。 |
| 建模/工艺人员 | 调整装配、对象变换、材质、图层、网格和有限精确特征。 |
| 测绘/点云人员 | 查看超大点云、分类着色、框选、裁剪、测距、配准辅助。 |
| 二次开发人员 | 通过插件/API 注册导入器、命令、属性面板、分析任务与导出器。 |

## 4.2 版本范围

| 阶段 | 范围 |
|---|---|
| P0 / MVP | 项目创建/打开；DXF、STEP、PCD、glTF/GLB 导入；DWG 只读试用链路；图层/树、相机、选择、测量、剖切、对象变换、材质基础编辑；命令事务、撤销重做、渐进加载、崩溃恢复。 |
| P1 / 正式 V1 | DWG 正式 SDK；高保真文字/块/填充；点云八叉树；网格顶点/边/面编辑；批量选择；属性系统；导出 GLB/DXF；插件 SDK；性能诊断。 |
| P2 / CAD 增强 | 精确草图、拉伸/旋转/布尔/倒角/圆角、装配约束、特征历史、STEP 写出、版本比较。 |
| 暂不承诺 | 完整 AutoCAD 命令兼容、全格式无损往返、复杂 NURBS 曲面设计、多人实时协同、云端渲染。 |

## 4.3 核心导入流程

1. 用户通过 Electron 文件对话框选择源文件，主进程返回受控文件句柄或令牌。
2. NestJS 创建 `ImportJob`，读取头部、计算 hash、选择格式适配器并发布进度。
3. Worker 或原生侧车解析源数据，同步生成文档实体、精确引用和首屏显示代理。
4. 几何分块写入 CAS；SQLite 提交文档实体、引用、版本和任务状态。
5. Renderer 先加载 manifest 与粗 LOD，再根据相机和屏幕误差逐步替换。
6. 导入完成后生成索引、缩略图、BVH、点云层级和诊断报告。

## 4.4 核心编辑流程

1. 用户操作先在 renderer 创建临时 preview，不修改持久化文档。
2. 用户确认后生成 `CommandEnvelope`，携带 `base_revision` 和事务 ID。
3. NestJS 校验权限、版本、实体存在性和资源预算。
4. 轻命令直接提交；重命令进入 Worker 或 Native Sidecar。
5. 事务成功后 revision 增长并发布资源失效事件；失败则保持原 revision。
6. renderer 只更新受影响的投影、缓冲区和属性面板。

## 4.5 核心撤销流程

- 撤销和重做均以新命令实现，不篡改历史记录。
- 轻量变换使用逆命令；局部网格修改使用稀疏 patch；拓扑/B-Rep 操作使用版本化资源或检查点。
- CAS 中旧资源按引用计数和历史保留策略延迟回收。
