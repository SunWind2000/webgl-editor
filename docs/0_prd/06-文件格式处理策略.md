# 6. 文件格式处理策略

## 6.1 总体矩阵

| 格式 | 阶段 | 路线 | 实现要点 | 主要风险 |
|---|---:|---|---|---|
| glTF/GLB | P0 | 前端直接加载；大文件可后台优化 | `GLTFLoader` + Meshopt/Draco/KTX2；保留 node/mesh/material 元数据 | 不作为精确 CAD 主存格式 |
| DXF | P0 | JS Worker 快速路径 + 原生高保真路径 | 常见 LINE/ARC/CIRCLE/LWPOLYLINE/TEXT/INSERT/HATCH；按图层/块实例批处理 | 字体、填充、代理对象和回写 |
| DWG | P1 正式 | 商业 SDK | ODA Drawings SDK 或 RealDWG；解析放原生侧车 | 授权、版本覆盖和平台矩阵 |
| STEP | P0 | OCCT 原生侧车 | STEP/XDE 装配、名称、颜色、层；Shape Healing；按弦差/角差三角化 | 解析和三角化耗时，需渐进结果 |
| PCD | P0 | 小文件直读；大文件预处理 | 支持常见字段；转八叉树/瓦片、量化坐标、分层抽样 | 原始 PCD 不提供大规模流式层级 |

## 6.2 glTF/GLB

- 作为显示资产和跨工具实时交换格式。
- 支持 Draco、Meshopt、KTX2/Basis 等常用压缩扩展。
- 导入后保留节点层级、材质、动画、实例和自定义 metadata。
- 大型 GLB 可在后台拆分为内容寻址 chunk，以便局部加载和缓存。
- glTF/GLB 不承载 STEP/DWG 的精确拓扑语义，不替代 B-Rep 主资产。

## 6.3 DXF

采用双路径：

### 快速路径

- 在 WebWorker 中解析 ASCII/Binary DXF 的常用 section 和实体。
- 将线、圆弧、多段线等转换为批处理 typed arrays。
- 块引用尽量保留实例，不展开为重复几何。
- 首屏优先生成粗略显示代理，复杂填充和文字异步细化。

### 高保真路径

- 使用 DWG/DXF 商业 SDK 或原生解析器。
- 处理复杂字体、HATCH、块属性、代理对象、坐标系和高版本兼容。
- 需要高保真回写时，必须保留源数据库对象与稳定 ID 映射。

## 6.4 DWG 许可与技术决策

- **ODA Drawings SDK：** 适合跨平台商业产品路线，需评估会员/发布许可、版本覆盖和原生包体。
- **Autodesk RealDWG：** 强调与 Autodesk DWG 兼容，需评估平台、分发和商业条款。
- **GNU LibreDWG：** 采用 GPLv3+；闭源商业产品直接链接或分发前必须完成法务与架构评估，不建议作为默认主线。
- 不建议以纯 JavaScript 第三方库作为生产级 DWG 主路径。

## 6.5 STEP 双轨模型

1. 读取 STEP/XDE，构建装配、实例、颜色、层和源实体映射。
2. 对 B-Rep 执行有效性检查和可配置 Shape Healing。
3. 先生成粗 LOD 返回首帧，后台再生成标准/精细 LOD 与边线。
4. 原始 STEP 与序列化 B-Rep 作为精确资产；显示网格按参数可重建。
5. 精确测量和建模命令发送给原生内核，结果产生新 B-Rep 资源版本。
6. 三角化必须输出拓扑 face/edge 到 primitive range 的映射，以支持选择、测量和编辑。

## 6.6 大规模点云专项

- 导入时建立八叉树/HLOD；每节点记录 AABB、点数、误差、属性范围和子节点。
- 渲染器按屏幕空间误差和全局 point budget 选择节点，超预算时先降级远处节点。
- 坐标局部量化为 `uint16`/`uint24`，tile origin 使用 float64；颜色、强度和分类列式存储。
- 选择与裁剪先在节点级粗筛，再在 Worker 中做点级过滤。
- PCD 作为源格式保留，内部运行格式使用分层、分块、可随机读取的点云瓦片。

## 6.7 格式适配器接口

```ts
interface FormatAdapter {
  readonly id: string;
  readonly version: string;
  probe(input: FileToken, head: Uint8Array): Promise<ProbeResult>;
  import(request: ImportRequest, sink: ImportSink): Promise<ImportResult>;
  export?(request: ExportRequest): Promise<ExportResult>;
  capabilities(): FormatCapabilities;
}
```

每次导入必须记录适配器版本、参数、源 hash、告警、失败阶段和派生资源 profile，确保缓存可失效重建。
