# 15. 风险清单与待决事项

## 15.1 风险清单

| ID | 风险 | 等级 | 缓解 |
|---|---|---:|---|
| R-01 | DWG 商业许可/平台限制 | 高 | 立项即完成 ODA/RealDWG POC、报价、分发条款和 OS 矩阵。 |
| R-02 | “完整建模”范围膨胀 | 高 | 将场景/网格编辑、精确 B-Rep、参数化 CAD 分层定义并独立验收。 |
| R-03 | WebGPU 生态兼容 | 中高 | WebGL2 为基线；封装 RendererAdapter；插件不得依赖后端私有 API。 |
| R-04 | 大模型内存复制 | 高 | chunk + accessor、Transferable/SAB、mmap，并监控 bytes copied。 |
| R-05 | 实体 ID 在三角化后丢失 | 高 | 从导入开始维护 source topology → primitive ranges 映射。 |
| R-06 | STEP/DWG 解析阻塞或崩溃 | 高 | 独立侧车、超时、资源上限、渐进结果和自动重启。 |
| R-07 | 撤销历史膨胀 | 中高 | 不可变资源、稀疏 patch、检查点、落盘和深度/预算策略。 |
| R-08 | SAB 部署失败 | 中 | 同源本地服务设置 COOP/COEP；能力检测；Transferable 降级。 |
| R-09 | 协议过度抽象 | 中 | 先以真实格式和编辑用例驱动；golden binary 与演进评审。 |
| R-10 | 跨平台原生构建 | 中高 | 固定工具链、可复现构建、签名、SBOM 和多 OS CI。 |
| R-11 | Vue/Three 状态耦合导致性能劣化 | 中高 | 领域模型、UI 状态和 GPU 资源严格分层，typed arrays 脱离响应式。 |
| R-12 | 点云内部格式选型过晚 | 中 | Phase 0 用真实数据冻结瓦片结构、量化和属性列式布局。 |
| R-13 | 商业 SDK 体积和升级影响发布 | 中高 | Sidecar 独立版本、延迟加载、签名和兼容层；商务条款前置。 |
| R-14 | 精确几何与显示代理状态不一致 | 高 | 所有代理带输入 hash、生成 profile、revision 和拓扑映射；可重建校验。 |

## 15.3 立项建议

先执行 4–6 周 Phase 0，不直接承诺“全格式完整建模”。POC 必须用真实 DWG/STEP/点云样本验证：格式保真、首屏时间、峰值内存、实体 ID 映射、撤销一致性、SDK 授权与跨平台发布。

只有以下条件全部满足，才进入正式 V1：

- DWG SDK 商务和技术路线冻结；
- 精确建模范围冻结；
- 基准样本和参考硬件冻结；
- 进程隔离、协议、命令事务和项目存储 POC 通过；
- WebGPU/WebGL2 和 SAB 降级链路验证通过。
