# 13. 非功能指标与验收

> 以下为 PRD 目标值，不是现有实现承诺；正式验收前应使用客户真实数据重新标定。

## 13.1 参考硬件与数据档位

| 档位 | 定义 |
|---|---|
| 基线工作站 | 6 核 CPU / 16 GB RAM / 独显 4 GB / NVMe / 1080p。 |
| 网格场景 S | ≤ 2M 三角形、≤ 5k 对象、纹理 ≤ 1 GB。 |
| 网格场景 M | ≤ 10M 三角形、≤ 50k 对象，允许 LOD/实例化。 |
| 2D 图纸 L | ≤ 2M 可见图元、≤ 20k 文本/块实例。 |
| 点云 L | 源数据 100M 点以上，视口可见点预算 2–8M。 |

## 13.2 性能目标

| ID | 类别 | 目标 |
|---|---|---|
| NFR-PERF-001 | 首屏 | manifest 打开后 1 秒内出现结构/包围盒；大模型 3 秒内显示粗 LOD 或明确进度。 |
| NFR-PERF-002 | 交互帧率 | 场景 S 目标 60 FPS；场景 M/点云 L 目标 ≥ 30 FPS；输入延迟 p95 < 50 ms。 |
| NFR-PERF-003 | 主线程 | 常规操作不得出现 >100 ms long task；解析、压缩和 BVH 必须异步。 |
| NFR-PERF-004 | 分块 | 单几何 chunk 4–32 MiB；每帧 GPU 上传字节数由 FrameScheduler 限制。 |
| NFR-PERF-005 | 内存 | Renderer/GPU/后端分别设预算；达到 80% 触发 LOD 降级和缓存回收。 |
| NFR-PERF-006 | 选择 | 10k 候选实体框选反馈 <100 ms；精确结果可在后续帧完成。 |
| NFR-PERF-007 | 取消 | 前端任务取消反馈 p95 <100 ms；原生长任务在安全检查点尽快终止。 |

## 13.3 格式验收

- 建立版本化格式样本库，包含正常、边界、损坏和超大文件。
- 每个文件记录源工具、格式版本、预期实体/装配/图层/颜色/属性。
- 导入结果生成差异报告：缺失实体、无法识别对象、三角化失败、字体替代和精度变化。
- DWG/STEP 的“成功”不能仅以文件打开为标准，必须验证语义映射和实体稳定 ID。

## 13.4 测试与验收矩阵

| 测试层 | 内容 | 验收 |
|---|---|---|
| 协议 | TS/C++ golden binary、旧版本读取、未知字段、畸形长度、64 位边界 | 同一输入 hash 一致；兼容矩阵通过 |
| 格式 | 官方/客户样本、装配、字体、HATCH、损坏文件、超大文件 | 样本库成功率与差异报告达标 |
| 命令 | 随机命令序列、事务失败、撤销重做、崩溃点注入 | 重放后文档 hash 一致 |
| 渲染 | WebGPU/WebGL2 截图、选择、剖切、透明、设备丢失 | 视觉差异阈值内，降级明确 |
| 性能 | 冷/热启动、首屏、帧时间、内存峰值、上传量、Worker 队列 | 各档位满足目标或有批准豁免 |
| 安全 | Electron 配置审计、路径穿越、恶意文件、fuzzing、端口访问 | 无高危问题，崩溃不越过隔离边界 |

## 13.5 命令一致性测试

- 生成随机但合法的命令序列，执行、撤销、重做后比较文档逻辑 hash。
- 在事务 prepare、WAL、CAS 写入、数据库 commit、事件发布等位置注入崩溃。
- 测试命令重复提交、网络超时重试、base revision 冲突和任务取消。
- 对 B-Rep 命令保存输入、参数、内核版本和输出 hash，便于回归定位。

## 13.6 性能基准要求

每次正式构建至少输出：

- 启动时间、项目打开、首屏和完整导入耗时；
- 主线程 p50/p95/p99 帧时间；
- DrawCall、三角形、可见点、GPU 上传量；
- renderer、NestJS、Worker、Sidecar 峰值内存；
- CAS 命中率、派生资源重建时间和 Worker 队列等待时间。
