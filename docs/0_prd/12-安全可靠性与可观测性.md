# 12. 安全、可靠性与可观测性

## 12.1 Electron 安全基线

- renderer 设置 `nodeIntegration=false`、`contextIsolation=true`、`sandbox=true`，只加载可信本地内容。
- preload 通过 `contextBridge` 暴露按业务命名的最小 API，禁止直接透传 `ipcRenderer`。
- 启用严格 CSP，禁止 `eval`、远程脚本和未签名插件；外部链接交由系统浏览器。
- 文件系统使用用户显式选择的句柄/令牌，路径做 canonicalize 和允许范围校验。
- Windows/macOS 发布包进行代码签名；自动更新验证签名并支持版本回退。

## 12.2 本地 HTTP 安全

- 仅绑定 `127.0.0.1` 随机端口，不监听局域网。
- 启动后由主进程传递端口和高熵 session credential。
- API 校验 Host、Origin、Content-Type、请求大小、session credential 和速率限制。
- 静态资源与 API 同源，以便统一 CSP、COOP、COEP 和凭证策略。
- 退出应用时主动关闭服务，残留进程由主进程监管清理。

## 12.3 原生解析器安全

- 原生解析器在隔离进程运行，设置文件大小、实体数、递归深度、解压比、CPU/内存和执行时限。
- 所有字符串长度、数组计数和 offset/length 做边界校验。
- 原生 parser 与 protobuf decoder 持续 fuzzing。
- 导入文件按不可信输入处理；崩溃、异常和超时不得越过进程隔离边界。
- 商业 SDK 与开源库建立 SBOM、版本锁定和漏洞升级流程。

## 12.4 可靠性要求

| ID | 要求 |
|---|---|
| NFR-REL-001 | 应用崩溃后最多丢失一个尚未提交事务；下次启动显示恢复入口。 |
| NFR-REL-002 | 原生导入器崩溃后任务标记失败，主窗口保持可用并保留诊断。 |
| NFR-REL-003 | 协议向后兼容至少覆盖同一 major 的最近三个 minor 写入版本。 |
| NFR-REL-004 | GPU 不支持 WebGPU 时自动回退 WebGL2，并在能力面板列出降级项。 |
| NFR-REL-005 | 项目迁移可中断、可备份、可回滚；禁止就地不可逆破坏。 |
| NFR-REL-006 | Worker/侧车重启后，幂等任务可自动重试，非幂等任务由事务层安全恢复。 |

## 12.5 可观测性

| 能力 | 要求 |
|---|---|
| 结构化日志 | `trace_id`、`document_id`、`job_id`、`command_id`、process、duration、bytes、error_code。 |
| 性能指标 | FPS、CPU frame、GPU frame、draw calls、triangles、GPU bytes、worker queue、cache hit。 |
| 任务追踪 | 导入各阶段耗时、实体计数、三角化参数、告警和重试。 |
| 资源指标 | CPU/RAM/GPU/共享 arena/CAS/缓存用量及预算压力。 |
| 诊断包 | 脱敏配置、版本、GPU 信息、最近日志、崩溃摘要；用户主动导出。 |

## 12.6 隐私与数据治理

- 默认离线运行，不上传工程文件、路径或模型内容。
- 遥测默认关闭或显式同意，且只采集脱敏性能和错误码。
- 诊断包生成前显示包含内容并允许用户排除日志或文件名。
- 最近文件、外部源路径和项目元数据按系统用户权限保护。

## 12.7 故障注入

验收阶段必须覆盖：

- renderer、Worker、NestJS、Native Sidecar 单独崩溃；
- 导入过程断电/强制退出；
- 磁盘满、无权限、源文件变化和 CAS 块损坏；
- GPU device lost、WebGPU 初始化失败和 WebGL2 回退；
- SSE 断线、请求超时、任务取消和重复提交。
