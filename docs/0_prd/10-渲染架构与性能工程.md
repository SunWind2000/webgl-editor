# 10. 渲染架构与性能工程

## 10.1 渲染后端策略

- WebGPU 为优先后端，WebGL2 为功能基线。
- 通过 `RendererAdapter` 封装材质、管线、后处理、能力查询和读回差异。
- 插件不得直接依赖后端私有 API，必须通过资源描述和能力接口注册。
- 首屏优先显示包围盒/粗 LOD，再增量替换；任何导入任务不得阻塞交互帧。

## 10.2 渲染内核模块

| 模块 | 职责 |
|---|---|
| RendererAdapter | WebGPU/WebGL2 统一能力和资源接口。 |
| SceneProjection | EngineDocument 到 Three.js 对象/批处理结构的投影。 |
| ResourceManager | CPU/GPU 资源生命周期、引用、LRU、设备丢失恢复。 |
| LODManager | 视锥、屏幕误差、距离、重要度和预算驱动 LOD。 |
| PickingSystem | GPU 粗选、BVH 精选、实例和拓扑映射。 |
| OverlaySystem | 测量、标注、操纵器、坐标轴、网格和选择轮廓。 |
| FrameScheduler | 每帧 CPU 工作、GPU 上传和资源创建预算。 |

## 10.3 场景组织与 DrawCall 控制

- 二维线段按图层、线型、线宽和可见性批处理，避免每条线一个 `Object3D`/DrawCall。
- 块引用、重复零件、点标记和符号使用 instancing；属性与选择状态映射到实例缓冲。
- 大型网格按材质、空间和实体映射拆成可寻址 chunk，支持局部卸载与选取。
- 属性数据优先 Structure-of-Arrays/typed arrays；Vue 响应式只保存轻量 UI 状态和引用。
- 文字使用字形图集/SDF 或批量排版，不为每个字符创建独立 DOM/Canvas 纹理。

## 10.4 选择

- 粗选使用 GPU ID buffer、实例 ID 或空间索引。
- 精确命中使用 Mesh BVH 或原生拓扑查询。
- 选取结果返回稳定实体 ID、primitive range、实例 ID 和可选拓扑 face/edge ID。
- 框选/套索先返回渐进粗结果，精确结果可在后续帧补齐。
- 海量选择状态使用 bitset/selection texture，而不是逐对象切换材质。

## 10.5 剖切与隐藏线

- 交互剖切优先在 shader 实现。
- 需要截面轮廓时在 Worker/Native Sidecar 异步生成并缓存。
- 隐藏线可采用深度 pre-pass + edge pass，P0 不追求 CAD 打印级完全一致。
- 透明对象需限制排序成本，超大场景可按分区近似排序或使用明确降级策略。

## 10.6 大坐标与精度

- 世界坐标、工程原点和文档变换保留 float64。
- 上传 GPU 前转为相机附近局部 float32，并记录 tile/chunk origin。
- 相机跨越阈值时执行 floating origin/rebasing，避免抖动。
- 拾取和测量结果回到 float64 文档坐标计算。

## 10.7 帧调度与内存预算

| 预算域 | 策略 |
|---|---|
| 主线程 | 每帧执行固定毫秒预算；长任务拆分，常规操作不得出现 >100 ms long task。 |
| GPU 上传 | FrameScheduler 限制每帧 bytes uploaded 和资源创建数量。 |
| GPU 内存 | 纹理/几何 LRU、可见性 pin、设备丢失恢复；达到 80% 预算时降级。 |
| Worker 内存 | 任务申报 `estimated_memory`；超预算排队、拆分或转后台。 |
| 缓存 | 原始/精确资产不可自动删除；LOD、BVH、缩略图等派生数据可重建。 |

## 10.8 渐进加载策略

1. 加载 manifest、结构树和包围盒。
2. 加载可见区域粗 LOD。
3. 按相机、屏幕误差和用户选择提升 LOD。
4. 后台生成/下载边线、BVH、点云属性和精细材质。
5. 退出可见集的资源进入冷却队列，超过预算后卸载。

## 10.9 性能反模式

- 每个实体一个独立材质、几何和 DrawCall。
- 每帧遍历完整装配树或更新所有矩阵/包围盒。
- 在 Vue store 中深度监听 Three.js 场景与 typed arrays。
- 大量 `Array.from`、spread、JSON stringify 或结构化克隆。
- 在渲染循环内创建临时向量、矩阵和数组造成 GC 抖动。
