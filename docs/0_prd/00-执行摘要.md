# 0. 执行摘要

## 0.1 最终建议

采用**混合多进程架构**：

- Electron 负责桌面壳、安全边界、本地文件令牌、子进程监管和系统集成。
- Vue 3 + Three.js 负责编辑器交互、场景投影和实时渲染。
- NestJS 作为独立本地服务，负责项目、命令、缓存、持久化以及 HTTP/SSE。
- Open CASCADE Technology（OCCT）与 ODA Drawings SDK 或 Autodesk RealDWG 组成原生几何侧车，负责 STEP、DWG、B-Rep、精确测量和高成本三角化。
- 前后端图形数据采用“**Protobuf 元数据 + 分块二进制缓冲区**”；SSE 仅传任务进度和资源失效事件。

## 0.2 可行性判断

整体可行，但“完整三维建模”必须分层定义：

- 场景/网格级编辑、测量、剖切、标注、材质和对象变换可由 Three.js 生态实现。
- 精确实体建模、布尔、倒角、装配约束及 STEP 往返必须由 CAD Kernel 承担。
- Three.js 场景树不能作为唯一文档模型；撤销重做、持久化与版本控制必须建立在独立领域模型上。

| 能力 | 可行性 | 判断 |
|---|---:|---|
| 渲染与交互 | 高 | Three.js 可覆盖 glTF、常规网格、线框、选择与编辑器交互；WebGPU 优先，WebGL2 回退。 |
| DXF | 中高 | 常用实体可走 JS/Worker 快速路径；文字、填充、块、代理对象及高保真回写需原生 SDK。 |
| DWG | 中 | 技术上可行，但商业 SDK 授权、版本兼容、平台矩阵和分发条款是项目门槛。 |
| STEP / B-Rep | 高（需原生） | OCCT 可用于 STEP/XDE、装配属性、Shape Healing、三角化及精确运算。 |
| 大规模点云 | 中高 | 需要离线或后台八叉树/HLOD 预处理，不能只依赖直接 PCD 加载。 |
| 撤销重做 | 高 | 以领域命令日志、事务、逆命令/补丁和检查点实现；Three 场景不是唯一数据源。 |
| Worker / SAB | 高（需安全配置） | 建立优先级、依赖图、内存准入、取消、心跳与降级机制；SAB 需 cross-origin isolation。 |

## 0.3 立项前必须冻结的决策

1. DWG 采用 ODA Drawings SDK 还是 Autodesk RealDWG。
2. V1“编辑”是否包含精确 B-Rep 特征；若包含，明确命令清单。
3. 目标数据规模、最低 GPU、操作系统、客户样本和可接受导入耗时。
4. 项目是目录包还是单文件容器，以及源文件是否内嵌。
5. 插件是否允许原生代码，以及签名、权限和沙箱机制。

## 0.4 推荐推进方式

先执行 4–6 周 Phase 0 架构 POC，不直接承诺“全格式完整建模”。POC 必须使用真实 DWG、STEP、点云和 GLB 数据验证：

- 格式保真与装配/图层/颜色/文字映射；
- 首屏时间、导入时间、峰值内存和崩溃隔离；
- 源实体、拓扑面、渲染 primitive 与选取 ID 的稳定映射；
- 命令事务、撤销恢复和项目重放的一致性；
- DWG SDK 授权、包体、签名和跨平台发布链路。
